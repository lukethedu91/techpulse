Linux Terminal Commands You'll Actually Use

The terminal isn't intimidating once you know a couple dozen commands. Here are the ones that actually matter, with real examples instead of dry syntax descriptions.

By Staff Writer  ·  March 2026  ·  9 min read

Most terminal guides try to be encyclopedias. They document every flag for every command, teach you the history of Unix, and spend three pages on man pages before you've done anything useful. This isn't that.

This is the subset of commands I actually type on a regular basis, explained in terms of what they do rather than what the man page says about them. If you work through all of these and understand when to reach for each one, you'll handle 90% of the tasks that send people to Stack Overflow.

Navigation and Files

pwd — tells you where you are. "Print working directory." Run it when you're confused about your location in the filesystem.

ls -la — lists everything in the current directory including hidden files (-a) in long format (-l) which shows permissions, owner, size, and modified date. ls alone works but ls -la is what you usually want.

cd — change directory. cd /etc takes you to /etc. cd ~ takes you home. cd - takes you back to where you just were — surprisingly useful.

mkdir -p some/nested/path — creates a directory and any parent directories that don't exist yet. Without -p, it fails if the parent doesn't exist.

cp -r source destination — copies files. Add -r to copy directories recursively.

mv source destination — moves files, or renames them if source and destination are in the same directory.

rm -rf directory — deletes a directory and everything in it. There is no undo. No trash bin. Be sure before running this.

Viewing File Contents

cat filename — dumps the whole file to the terminal. Good for short files, unusable for long ones.

less filename — opens the file in a pager you can scroll through. Navigate with arrow keys or j/k, search with /searchterm, quit with q.

tail -f logfile — streams the end of a file as it's written to. Essential for watching log files in real time. Ctrl+C to stop.

head -n 20 filename — shows the first 20 lines. Useful for checking what format a large file starts with.

Searching

grep "pattern" filename — finds lines matching a pattern. Add -r to search recursively through directories, -i to ignore case, -n to show line numbers.

grep -rn "TODO" ./src/   # find all TODOs in your source code

find /path -name "*.log" -mtime -7 — finds files matching a pattern, modified in the last 7 days. find has a strange syntax but is very powerful once you know it.

"Piping is the thing that makes the terminal feel like a superpower. Small tools, chained together."

Pipes and Redirection

The pipe | sends the output of one command as input to another. This is where terminal fluency pays off:

cat access.log | grep "404" | wc -l   # count 404 errors in a log
ps aux | grep nginx                    # find nginx processes
ls -la | sort -k5 -n                   # sort directory listing by file size

> redirects output to a file (overwrites). >> appends. 2> redirects stderr.

command > output.txt 2>&1   # redirect both stdout and stderr to a file

Process Management

ps aux — lists all running processes. Pipe to grep to find a specific one.

kill PID — sends a signal to a process. kill -9 PID is the forceful version that can't be caught or ignored. Get the PID from ps aux.

htop — an interactive process viewer, much better than top. Install it if it's not there (sudo apt install htop). Navigate with arrow keys, kill processes with F9.

Permissions

chmod 755 filename — changes file permissions. 755 is "owner can read/write/execute, everyone else can read/execute." 644 is "owner can read/write, everyone else can read." 600 is "owner only."

chown user:group filename — changes file ownership. sudo chown -R www-data:www-data /var/www/html is a common pattern for web server files.

SSH and Remote Work

ssh user@hostname — connect to a remote machine. If you've set up SSH keys, no password prompt.

scp localfile user@host:/remote/path — copy a file to a remote machine. rsync -avz is better for directories and handles resuming.

screen or tmux — terminal multiplexers that let you run sessions that persist even when you disconnect from SSH. Learn one of these if you run anything long-running on remote servers.
